# Customer portal — white-label setup

One **agency per deployment**. Branding, navigation, copy, and features are configured at **build time** via environment variables or a JSON preset.

## Quick start (First Cargo)

```bash
cp .env.example .env.local
npm install
npm run dev   # http://localhost:3002
```

## Configuration layers

1. **Defaults** — `lib/portal-config/defaults.ts`
2. **Env vars** — simple overrides (brand, support, feature flags)
3. **JSON preset** — `NEXT_PUBLIC_PORTAL_CONFIG_JSON` for full control (nav tabs, drawer, quick actions, copy)

Example preset: [`config/portal.first-cargo.json`](config/portal.first-cargo.json)

### Apply a JSON preset locally

```bash
# One line (minified) in .env.local:
NEXT_PUBLIC_PORTAL_CONFIG_JSON='{"appName":"First Cargo","navigation":{"bottomTabs":[{"id":"home","href":"/dashboard","label":"Accueil","icon":"House"}]}}'
```

Or load from file when building Docker:

```dockerfile
ENV NEXT_PUBLIC_PORTAL_CONFIG_JSON="$(cat config/portal.first-cargo.json)"
```

## Environment variables

| Variable                                | Purpose                                                        |
| --------------------------------------- | -------------------------------------------------------------- |
| `NEXT_PUBLIC_AGENCY_SLUG`               | Backend agency scope (`x-customer-agency-slug`)                |
| `NEXT_PUBLIC_APP_NAME`                  | Fallback app title                                             |
| `NEXT_PUBLIC_BRAND_PRIMARY`             | Primary color (`#263070`)                                      |
| `NEXT_PUBLIC_BRAND_LOGO_URL`            | Header / drawer logo                                           |
| `NEXT_PUBLIC_PORTAL_NOTICE`             | Home screen banner text                                        |
| `NEXT_PUBLIC_SUPPORT_PHONE`             | Support tel link                                               |
| `NEXT_PUBLIC_SUPPORT_EMAIL`             | Support email                                                  |
| `NEXT_PUBLIC_LOCATIONS_URL`             | External locations page                                        |
| `NEXT_PUBLIC_STAFF_APP_URL`             | Link to staff operations app                                   |
| `NEXT_PUBLIC_PORTAL_VERSION`            | Shown loicassontia@gmail.comon account screen                  |
| `NEXT_PUBLIC_PORTAL_CONFIG_JSON`        | Full JSON override (see types in `lib/portal-config/types.ts`) |
| `NEXT_PUBLIC_PORTAL_BOTTOM_NAV_VARIANT` | `classic` · `floating` · `pill` · `underline` · `solid`        |
| `NEXT_PUBLIC_PORTAL_SHOW_NOTIFICATIONS` | `true` / `false`                                               |
| `NEXT_PUBLIC_PORTAL_SHOW_DRAWER`        | Side menu                                                      |
| `NEXT_PUBLIC_PORTAL_SHOW_POWERED_BY`    | “Powered by NTIGI” in drawer                                   |
| `NEXT_PUBLIC_PORTAL_SHOW_QUICK_ACTIONS` | Home action buttons                                            |
| `NEXT_PUBLIC_PORTAL_SHOW_SHIPMENT_LIST` | Home shipment list                                             |
| `NEXT_PUBLIC_ENABLE_PWA`                | Enable PWA service worker + install prompt (`true` by default) |

## PWA (installable web app)

The customer portal ships as an installable **PWA** with offline fallback and an install prompt on supported browsers.

```bash
# Manifest is generated from env at build time
npm run build

# Disable locally if you need to debug without a service worker
NEXT_PUBLIC_ENABLE_PWA=false npm run dev
```

`public/manifest.json` is generated by `scripts/generate-pwa-manifest.mjs` using `NEXT_PUBLIC_APP_NAME`, `NEXT_PUBLIC_BRAND_PRIMARY`, and `NEXT_PUBLIC_AGENCY_SLUG`.

## What you can customize in JSON

- **`navigation.bottomNavVariant`** — mobile tab bar style: `classic` (default), `floating`, `pill`, `underline`, `solid`
- **`navigation.bottomTabs`** — bottom bar (href, label, icon, matchPaths)
- **`navigation.drawerMenu`** — hamburger menu links
- **`navigation.legalLinks`** — terms, privacy
- **`home.quickActions`** — home buttons (`href`, `scroll`, `tel`, `external`)
- **`copy.*`** — UI strings (track hero, welcome, tab labels, etc.)
- **`features.*`** — toggle notifications, drawer, shipment list

Icons use Phosphor names: `House`, `MagnifyingGlass`, `User`, `Package`, `MapPin`, `Headset`, etc. (see `lib/portal-config/icons.tsx`).

## Code usage

```tsx
import { portalConfig, usePortalConfig } from "@/lib/portal-config";
// or
import { usePortalConfig } from "@/hooks/use-portal-config";
```

`portalConfig` is resolved once at build time — no runtime API call for layout/nav.

## API

Each deployment talks to the NTIGI backend with `NEXT_PUBLIC_AGENCY_SLUG` and the customer session cookie. Client shipments:

- `GET /clients/:id/shipments/outgoing` — sent parcels
- `GET /clients/:id/shipments/incoming` — received parcels
